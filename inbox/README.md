# inbox — 作業の受け取り窓口

モバイル等からの「やってほしいこと」を投げ込む場所。
ここにファイルがあれば、PC側で取り込んで実装に入る。

## 使い方（モバイル → ここ）

1. GitHubモバイルアプリ／Webで、このリポジトリの `inbox/` を開く
2. 「Add file」で **依頼を書いた `.md`** を1つ追加（画像があれば一緒に追加）
3. commit すると窓口に入る
4. PC側で `git pull` → 内容を読んで実装 → 終わったら `inbox/done/` へ移動

## 1依頼 = 1セットの置き方

**A. テキストだけの依頼** → `.md` を1ファイル置くだけ

```
inbox/2026-06-21_ripple-color.md
```

**B. 画像付きの依頼** → 依頼名のフォルダを作り、中に md と画像をまとめる

```
inbox/2026-06-21_new-bg/
├─ request.md      ← やってほしいこと
├─ ref1.png        ← 参考画像
└─ ref2.jpg
```

## ファイル名の規則

- 形式: `YYYY-MM-DD_短い英語スラッグ`（例: `2026-06-21_star-color`）
- 日付で並ぶので時系列が分かる。スラッグは内容が一目で分かる短い語。
- 厳密でなくてOK。迷ったら日付＋一言。

## 依頼MDのテンプレ（コピーして使う）

```markdown
---
title: <一言タイトル>
target: <対象サイト名 or 新規>   # 例: water-ripple / 新規
priority: 通常                   # 通常 / 急ぎ
---

## やってほしいこと
<具体的に。1〜数行でOK>

## 参考・補足
<画像の説明、URL、好みの色やトーンなど。なければ空でOK>
```

## 状態の扱い

- `inbox/` 直下にある md・フォルダ = **未処理（キュー）**
- 処理が終わったもの = `inbox/done/` に移動（履歴として残す）
- このREADMEと `done/` 自体は処理対象ではない

## 注意

- 窓口はリポジトリに含まれる（追跡対象）。機密情報は置かない。
- 画像は依頼に必要な参考用のみ。大量・大容量は避ける。
- 自動処理の構想は [docs/inbox自動処理_検討案.md](../docs/inbox自動処理_検討案.md) を参照。
