# 動的デザインツール 技術要件（本格プロジェクト化）

最終更新: 2026-06-13
対象: 水面リップル / 星の軌跡 などのジェネラティブ・インタラクティブ表現を「量産・配布できる本格プロジェクト」へ拡張する

---

## 0. 現状と目標

**現状（雛型）**
- 単体 `index.html` に CSS/JS をベタ書き。Canvas2D / WebGL1 を手書き。
- 設定パネルも手書き。ビルド工程・テスト・バージョン管理なし。
- 作品ごとにコピー＆改変で増殖 → 共通部分が重複し、保守しづらい。

**目標（完成品）**
- 共通の描画エンジンの上に「エフェクト」を差し替える構造にし、新作を短時間で量産。
- 高品質化（WebGL2 / ポストプロセス）、書き出し（動画・高解像度静止画）、プリセット共有。
- Git 管理・自動テスト・自動デプロイで「製品」として配布できる状態にする。

## 1. プログラミング技術 / ライブラリ（コア）

### 1.1 レンダリング基盤
- **WebGL2**（標準化の本命。GLSL ES 3.0、複数レンダーターゲット、より良いテクスチャ）。将来的に **WebGPU**（WGSL、計算シェーダーで波紋シミュ等が高速化）。
- ライブラリ選定:
  - **Three.js** … 汎用3D・シーン/カメラ/ポストプロセス一式。最有力。
  - **OGL** … 軽量WebGL（薄いラッパー。シェーダー主体の2D背景に好相性）。
  - **regl** … 関数型WebGL。ピンポンFBO（現行水面の手法）をきれいに書ける。
  - **PixiJS** … 2D特化で高速（多数のスプライト/パーティクル向き）。
  - **p5.js** … 学習・プロトタイプ用（本番より試作向き）。
  - 推奨: 背景エフェクト主体なら **OGL or Three.js**。星の軌跡のような点群は **PixiJS** も候補。

### 1.2 シェーダー
- **GLSL ES 3.0**（WebGL2）。共通シェーダー部品は **lygia**（ノイズ/ブラー/カラー等の定番関数群）、ビルド連携は **glslify**。
- ノイズ: simplex / curl noise（自然な揺らぎ・流れ）。
- 現行のピンポンFBO波紋を「再利用可能なシミュレーションモジュール」として切り出す。

### 1.3 ポストプロセス（品質の鍵）
- **postprocessing**（Three.js対応）or 自前パス: **Bloom（発光）**, FXAA/SMAA（アンチエイリアス）, ビネット, 色収差, トーンマッピング。
- 参考デモの「キラキラ」感は Bloom 由来が大きい。星の軌跡・水面どちらも効果大。

### 1.4 アニメーション / タイムライン
- **GSAP**（堅牢なタイムライン・イージング。パラメータ遷移やイントロ演出に）。
- 代替: anime.js。時間駆動のパラメータ制御を一元化する。

### 1.5 コントロールUI（手書きパネルの置き換え）
- **Tweakpane**（推奨）または **lil-gui**。
  - スライダー/カラーピッカー/フォルダ/モニタ（FPS等）を自動生成。
  - **プリセットの保存・読み込み**、パラメータの JSON 入出力が容易。
  - 現状の手書き `.panel` を廃し、エフェクトのパラメータ定義から自動生成へ。

### 1.6 数学・補助
- **gl-matrix**（行列/ベクトル）, **simplex-noise**, **chroma.js**（配色・グラデーション）, **stats.js**（FPS計測）。

### 1.7 入力・インタラクション
- Pointer Events に統一（マウス/タッチ/ペン共通）。ピンチ/ドラッグ等のジェスチャ。
- **prefers-reduced-motion** を尊重（モーション過敏配慮）。

### 1.8 パフォーマンス
- **OffscreenCanvas + Web Worker** で描画をメインスレッドから分離。
- devicePixelRatio 制御、解像度スケーリング、**適応品質**（FPS低下時に解像度/星数を自動調整）。
- Page Visibility API でバックグラウンド時に停止。

### 1.9 書き出し・録画（“作品”として配布）
- **MediaRecorder** / **CCapture.js** で WebM/MP4/GIF 録画。
- 高解像度静止画書き出し（オフスクリーンで大判レンダー）。**html2canvas** はUI込みキャプチャ用。

### 1.10 オーディオ反応（任意の拡張）
- **Web Audio API** / **Tone.js** で音に反応する波紋・星。

### 1.11 永続化・共有
- JSON プリセット、**URLハッシュにパラメータを載せて共有**、localStorage で前回設定を復元。
- ギャラリー/プリセット共有をクラウド化するなら **Supabase**（DB/ストレージ/認証）。

## 2. 開発基盤 / ツールチェーン

- **Vite + TypeScript**（高速HMR、型安全）。パッケージ管理は **pnpm**。
- **ESLint + Prettier + EditorConfig**（整形・静的解析）。
- ディレクトリ構成（案）:
  ```
  src/
    engine/    … レンダーループ・リサイズ・入力・FBO・時間管理（共通）
    effects/   … ripple/ startrail/ … 各エフェクト（共通interfaceを実装）
    ui/        … Tweakpane連携・プリセット
    export/    … 録画・静止画書き出し
    presets/   … JSON
  ```
- **テスト**: Vitest（ロジック）、**Playwright**（ビジュアルリグレッション＝スクショ差分で「四角バグ」類の再発を検知）。
- **バージョン管理**: Git / **GitHub**（PRレビュー・Issue・履歴）。
- **CI/CD**: GitHub Actions（lint → test → build → deploy）。
- **デプロイ**: **Vercel** または **Netlify**（プレビューURL自動発行）、軽量なら GitHub Pages。

## 5. プラグイン

- プラグイン検索結果は大規模で、本件（汎用クリエイティブコーディング）に直結する**組織特化プラグインは限定的**でした。

## 6. アーキテクチャ提案（エフェクト・プラグイン型エンジン）

- **共通 Engine**: レンダーループ / リサイズ / 入力 / FBO / 時間。
- **Effect インターフェース**: `init() / update(dt) / render(t) / resize() / dispose()` ＋ `params` 定義（スキーマ）。
- パラメータスキーマから **Tweakpane を自動生成**し、プリセット保存と直結。
- **ギャラリー**: 複数エフェクトをドロップダウンで切替（水面・星の軌跡・今後の新作）。
- これにより新作は「effects/ に1ファイル追加」で量産可能になる。

## 7. ロードマップ（雛型 → 完成品）

1. **Phase 1 — 基盤化**: Vite+TS プロジェクト化。既存2作品を移植し、共通 Engine を抽出。Git/GitHub 開始。
2. **Phase 2 — 操作性**: Tweakpane 化、プリセット保存・URL共有、書き出し（動画/静止画）。
3. **Phase 3 — 品質**: WebGL2 + ポストプロセス（Bloom 等）で参考デモ級の質感へ。
4. **Phase 4 — 運用**: Vitest/Playwright・CI・自動デプロイ。ギャラリー公開。
5. **Phase 5 — 拡張**: 新エフェクト量産、オーディオ反応、AI背景生成の組込み。

---

## 8. 品質・非機能要件

- 60fps 目標（低スペック時は適応品質）。モバイル/タッチ対応。
- prefers-reduced-motion 対応、色のコントラスト等アクセシビリティ。
- ライセンス整理（利用OSSのライセンス、フォント、AI生成画像の権利）。
- 既存の注意点を継承: `file://` で画像を使う場合の CORS 回避（data URL 化）、ファイル編集はファイルツール経由で（マウントの古いキャッシュに注意）。

## 付録: 最小スタック vs 推奨スタック

| 項目 | 最小（手軽） | 推奨（本格） |
|---|---|---|
| 構成 | 単体HTML + CDN | Vite + TypeScript + pnpm |
| 描画 | 手書きWebGL/Canvas | OGL/Three.js + WebGL2 |
| UI | 手書きパネル | Tweakpane / lil-gui |
| 効果 | 自前 | postprocessing（Bloom等） |
| 録画 | なし | MediaRecorder / CCapture |
| テスト | 目視 | Vitest + Playwright |
| 配布 | ファイル共有 | GitHub + Vercel/Netlify |
