# inbox 自動処理 — 検討案

最終更新: 2026-06-21
ステータス: **構想のみ（未実装）**

`inbox/` に投げ込まれた依頼を、自動でチェックして実装に入る仕組みの検討メモ。
まずフォルダと規則だけ用意済み。自動化は本書を土台に後日導入する。

---

## 1. 目的

モバイルから `inbox/` に「依頼MD＋画像」を commit するだけで、
PC側（Cowork）が定期的に窓口を見に行き、未処理があれば取り込んで実装まで進める。
手作業の「pullして読んで着手」を自動化する。

## 2. 全体フロー（想定）

```
[モバイル] inbox/ に依頼をcommit
      │
      ▼
[自動チェック] 定期実行で git pull → inbox/ に未処理があるか確認
      │  未処理なし → 何もせず終了
      │  未処理あり ▼
[取り込み] 依頼MDと画像を読む
      │
      ▼
[実装] 内容に沿って sites/ 等を変更
      │
      ▼
[後処理] 依頼を inbox/done/ へ移動 → commit & push
      │
      ▼
[通知] 完了内容をユーザーに報告
```

## 3. 検知ロジック

- **未処理キュー** = `inbox/` 直下にある `.md` ファイル、および依頼フォルダ
  （`README.md` と `done/` は除外）
- **処理済み判定** = `inbox/done/` 配下に移動されているか
- これにより「何が残っているか」がフォルダを見るだけで分かる（状態管理が不要）

## 4. 実現方式の比較

| 方式 | 検知 | 実装まで自動 | 備考 |
|------|------|:----------:|------|
| **A. Coworkスケジュールタスク（推奨）** | 定期実行でpull＆確認 | ○ | Claudeが読んで実装・push・通知まで一気通貫。今すぐ作れる |
| B. GitHub Actions | pushで即時 | △ | 通知やCIには強いが、Claudeでの「実装」は別途連携が必要 |
| C. ローカル監視スクリプト | フォルダ監視で即時 | △ | PCが起動中のみ。検知→通知止まりになりがち |

→ 当面は **A（Coworkスケジュールタスク）** が最も素直。即時性が要るなら将来Bを併用。

## 5. 推奨方式A：スケジュールタスクのプロンプト案

定期タスク（例：毎朝8時 / 1日数回）に、おおよそ次の指示を持たせる想定。

```
1. リポジトリで `git pull` し、inbox/ 直下を確認する
   （README.md と done/ は無視）
2. 未処理の依頼が無ければ、何もせず終了する
3. 未処理があれば、各依頼のMDと付随画像を読む
4. 内容に沿って該当サイト（sites/ 配下）を実装・修正する
5. 一覧トップ index.html への追記が要れば行う
6. 完了した依頼一式を inbox/done/<依頼名>/ へ移動する
7. 変更を commit して push する
8. 実施内容（何をどう変えたか）を要約して報告する
```

## 6. 安全策（重要）

- **公開反映の確認**: 変更は即 push → Pages に公開される。重要な変更や曖昧な依頼は、
  自動実装せず「内容を要約して確認を求める」段階で止める運用にする。
- **機密を置かない**: inbox はリポジトリに含まれ公開リポジトリにあるため、
  個人情報・鍵・パスワード等は投げない（READMEにも明記済み）。
- **破壊的変更の抑制**: 既存サイトの削除・大改変は自動では行わず、必ず確認を挟む。
- **冪等性**: done/ 移動済みは再処理しない。途中失敗時は done/ に動かさず残す。

## 7. 段階的な導入プラン

1. **第1段** 通知のみ … 未処理があれば「依頼が来ています」と要約報告だけ（実装はしない）
2. **第2段** 半自動 … 軽微な依頼（色変更・文言修正など）は自動実装、要確認は止める
3. **第3段** 自動 … 信頼できる範囲を広げ、done/移動＋pushまで自動化

まず第1段で運用感を確かめ、誤作動が無いことを確認してから範囲を広げる。

## 8. 今後の検討事項（未決）

- 実行頻度（毎朝 / 数時間ごと / 任意タイミング）
- 「急ぎ」依頼の扱い（priority: 急ぎ を見て即時通知するか）
- 失敗・判断保留時の通知先と書式
- done/ が増えた際の整理（一定期間で別アーカイブへ）
- 画像が重い場合の上限ルール
