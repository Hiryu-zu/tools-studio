# 自動化セットアップ

「スマホで inbox に置く → 自動で反映」に近づけるための仕組み。
役割を2つに分けている（この環境からは git が実行できないため）。

```
[スマホ] inbox に依頼をcommit
   │
   ▼
[Windows 同期.cmd（定期）] git pull   ← 依頼をPCへ取り込む
   │
   ▼
[Cowork 毎朝タスク] inbox を処理 → gallery/data.js 等に反映 → done/へ移動・報告
   │
   ▼
[Windows 同期.cmd（定期）] git add/commit/push  ← 変更を公開へ反映
   │
   ▼
[GitHub Pages] サイトに反映
```

## パート1：Cowork 毎朝タスク（設定済み）

- タスク名: `inbox-processor`
- 実行: 毎朝 7:10（ローカル時刻）
- 内容: inbox 直下の依頼を読み、ギャラリー追加などを自動実装 → `inbox/done/` へ移動 → 報告。
  大きい/曖昧な依頼は自動実装せず「要確認」に回す。機密は扱わない。
- 注意: **Coworkタスクはアプリが開いている間に実行**される。閉じていた場合は次回起動時に実行。
- 初回は、サイドバーの「Scheduled」から `inbox-processor` を一度「Run now」しておくと、
  必要なツール権限が事前承認され、以後の自動実行が途中で止まらない。

## パート2：Windows 同期.cmd（git pull/push の自動化）

リポジトリ直下の `同期.cmd` が `pull → add → commit → push` を実行する。
日本語パスを書かずに済むよう、自分の置かれた場所で動くようにしてある。

### 手動で使う場合

`同期.cmd` をダブルクリックするだけ。pull と push をまとめて行う。

### 定期実行（タスクスケジューラ）の設定

15〜30分ごとに自動同期したい場合：

1. スタートメニューで「タスク スケジューラ」を開く
2. 右の「基本タスクの作成」
3. 名前: `tools-studio 同期` → 次へ
4. トリガー: 「毎日」→ 次へ → 開始時刻を適当に → 次へ
5. 操作: 「プログラムの開始」→ 次へ
6. プログラム/スクリプト: `C:\便利＆ユニークツール制作\同期.cmd` を指定 → 次へ → 完了
7. 作成後、一覧で `tools-studio 同期` を右クリック →「プロパティ」→「トリガー」タブ →
   該当トリガーを編集 →「詳細設定」で「繰り返し間隔: 30分」「継続時間: 1日」にチェック → OK

これで30分ごとに pull/push が走る。Coworkタスクが朝に処理した内容も、次の同期で公開される。

## 全体の流れ（まとめ）

- スマホで inbox に「これ追加して」とURL付きでcommit
- 同期.cmd が pull（PCに取り込み）
- 翌朝（または手動Run now）に inbox-processor が gallery/data.js へ反映＋done/へ移動
- 同期.cmd が push → Pages に公開

## 急ぎで反映したいとき（手動でフル実行）

1. `同期.cmd` をダブルクリック（pull）
2. Coworkで inbox-processor を「Run now」
3. もう一度 `同期.cmd` をダブルクリック（push）
