## 2. 目標構成（モノレポ + 一覧トップページ）

スマホから **URL 1つ**で全成果物に辿り着ける構成にする。

```
便利＆ユニークツール制作/   ← リポジトリ root
├─ index.html              ← 【一覧トップ】全サイトへのリンク集（スマホ最適化）
├─ README.md               ← リポジトリの説明・運用ルールへのリンク
├─ フォルダ整理ルール.md    ← 本書
├─ .gitignore
│
├─ sites/                  ← 【Pages公開対象】動くサイトだけを置く
│   ├─ star-trail/             （星の軌跡アニメーション）
│   ├─ aspect-ratio-tool/      （画像比率変換ツール）
│   ├─ water-ripple/           （水面リップル背景エフェクト）
│   └─ generative-design-studio/  ← Viteのビルド成果(dist)を配置
│
├─ tools/                  ← 【非公開】ローカル実行ツール
│   └─ codex-usage-monitor/    （Claude-Codex残量モニタ）
│
├─ assets/                 ← 【非公開】サイトでない素材
│   └─ note-css-themes/        （ブラウザビジュアル）
│
└─ docs/                   ← メモ・仕様・参考リンク
    ├─ 技術要件_動的デザインツール.md
    └─ 資料.txt
```

ポイント:
- **`sites/` 以下だけが「動くウェブサイト」**。一覧トップ(`index.html`)からここへリンクする。
- `tools/` `assets/` `docs/` は履歴管理（バックアップ）はするが、トップページには載せない＝スマホ確認の対象外。
- Vite製は **ビルド済みの静的ファイルを `sites/generative-design-studio/` に置く**（ソースは別管理 or サブフォルダに `src-project/` として同梱し、`dist`成果だけ公開）。

## 3. 整理ルール（今後これに従う）

### ルール1: 置き場所は4分類で判断する
新しく作ったものは必ず次のどれかに入れる。迷ったら「ブラウザでそのまま動くか？」で判定。

- ブラウザで開いて動く → `sites/`
- PC上で実行するツール（Python/bat等）→ `tools/`
- 単体では動かない部品・素材 → `assets/`
- 文章・メモ・仕様 → `docs/`

### ルール2: フォルダ名は英小文字ハイフン区切り
URL になるため。日本語名は Pages の URL でエンコードされ崩れやすい。
表示名（日本語）は各サイトの `index.html` の `<title>` と一覧トップに書く。

- 例: `星の軌跡アニメーション` → フォルダ `star-trail/`、タイトルは日本語のまま
- 命名対応表は本書 or README に残す。

### ルール3: 各サイトは自己完結
`sites/<名前>/` の中だけで動くこと（外部の相対パス参照をしない）。
入口は必ず `index.html`。

### ルール4: コミットしないものを最初に決める（.gitignore）
重量物・自動生成物・実行時ファイルは追跡しない。

```gitignore
node_modules/
dist/            # ※Pages公開にdistを使う場合はサイト側のみ例外管理
*.local
.vite/
.DS_Store
Thumbs.db
state.json       # 監視ツールの実行時出力
usage_data.json
```

### ルール5: 各成果物に最低限のREADME
1行でいいので「何ができるか」を `README.md` に書く。3か月後の自分のため。

### ルール6: 不要ファイルは「迷ったら docs/_archive/ へ」
消すのが怖いもの（候補PNG、旧版）はゴミにせず `docs/_archive/` に退避。
ルートとサイト直下は常にきれいに保つ。

### ルール7: コミットは小さく、日本語1行でOK
例: `星の軌跡: 配色を調整` / `整理: water-rippleの未使用画像をarchiveへ`

## 4. GitHub Pages の方針

- リポジトリ Settings → Pages → Source: `main` ブランチ / `/ (root)`。
- ルートの `index.html` がトップページになり、`sites/<名前>/` へリンク。
- スマホ確認は **トップのURL 1つ**をブックマークすれば全部辿れる。
- Vite製サイトは事前に `npm run build` し、生成された静的ファイルを `sites/` 配下へ置く（Pagesはビルドしないため）。
