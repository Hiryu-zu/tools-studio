# 振り返り — イラスト回転トレイル（疑似スタートレイル）

最終更新: 2026-06-14
対象エフェクト: `src/effects/IllustrationTrailEffect.ts`
使用画像: `public/starfield.png`（4096×4096・白一色の同心円トレイル）

---

## 1. 概要

「軌跡を描き込んだ星空イラストを、極（=画像中心）まわりに回すだけ」で
疑似的なスタートレイルを表現する、**軽量・シンプル**な方式。
動的な軌跡計算（パーティクルや残像フィードバック）を使わず、
**回転とポストFXだけ**で成立させる。

- 軌跡はイラスト側に焼き込み済み → 実行時は回すだけ。
- 同心円構図なので回転に継ぎ目が出ない。
- 中心（極）は自然に静止し、外周ほど速く流れて見える＝本物の日周運動の挙動。

---

## 2. なぜこの方式か（3D実時間版との違い）

| 観点 | 3D星景（Bloom/パーティクル） | イラスト回転トレイル（本方式） |
|---|---|---|
| 仕組み | 加算パーティクル＋Afterimage＋Bloom | 軌跡入り画像を回す＋Bloom |
| 負荷 | 中〜高 | 低（テクスチャ1枚を回すだけ） |
| 画作り | 計算依存 | **イラストで完全に制御**できる |
| 軌跡 | 実時間生成 | 画像に焼き込み |
| 弱点 | 重い／調整が複雑 | 本質はループ。静止画感を消す工夫が要る |

ねらい: **実装をシンプルに保ちつつ、画はイラストで作り込む**。

---

## 3. 実装の仕組み（IllustrationTrailEffect）

- 直交カメラ（OrthographicCamera）の2Dシーン。
- **空**: 正方形テクスチャを貼った板（Plane）を、極の画面位置に置いて `rotation.z` で回す。
  - 回しても画面を覆うよう、**極から四隅までの最大距離の2倍**に板を拡大（オーバースキャン）。
    → 回転してもイラストの矩形エッジが画面に出ない。
  - 極は「画像中心」。中心を回すので**回転は完全に継ぎ目なし**（同心円は回しても見た目が変わらない）。
- **前景シルエット**（任意・静止）: 透過PNGを最前面に重ねて“本物の夜空”感を足せる。
- **ポストFX**: `UnrealBloomPass`（発光）＋ ACESトーンマッピング。Afterimageは不要なので不使用。
- 操作: 画面クリックで極の移動、パネルで 回転速度／輝き／露出／極X・Y／前景 を調整。

要点コード（擬似）:
```
plane.position = 極の画面座標
plane.scale    = (極→最遠コーナー距離) × 2 × 1.02   // 画面を必ず覆う
each frame: plane.rotation.z += speed * dt
composer = RenderPass → UnrealBloom → OutputPass
```

## 4. 画像の作り方（重要）

最終的に使ったのは **Python(Pillow) によるプロシージャル生成**。
回転前提なので「正方形・極が中央・同心円・白一色・くっきり」を満たすよう作った。

### 手順（実際に使ったロジック）
1. **2倍解像度(8192²)で描いて、最後に4096²へ縮小**（LANCZOS）。
   → PILの円弧はアンチエイリアスが弱いので、縮小でAAをかけて滑らか＆くっきりに。
2. 背景は暗い青 `(6,9,22)`＋中心のごく淡いグロー（小さなグラデを拡大してブロックを防ぐ）。
3. **同心円の軌跡**を多数(約9000本)描く。各弧は:
   - 半径 `r = sqrt(乱数) × 最大半径`（面積一様＝外周ほど密に）。
   - 弧長は外周ほど長め。開始角はランダム。
   - 色は**白のみ**、明るさだけ変化（`v = 120〜255`）。
   - 各弧は「太く暗いグロー下地」→「本線」の二度描きで、ほのかな滲みを付与。
4. 中心に**静かな極の領域**＋小さな**極星**（白点）。
5. 4096²でPNG保存し `public/starfield.png` を上書き。

### 生成スクリプトの核（抜粋）
```python
from PIL import Image, ImageDraw
import math, random
S = 8192; cx = cy = S/2; maxr = int(S*0.74)
img = Image.new('RGB', (S, S), (6, 9, 22))
d = ImageDraw.Draw(img)
def arc(r, a0, a1, v, w):
    d.arc([cx-r, cy-r, cx+r, cy+r], a0, a1, fill=(v, v, v), width=w)
for _ in range(9000):
    r = math.sqrt(random.random()) * maxr
    if r < 30: continue
    span = random.uniform(5, 22) * (0.5 + r/maxr)
    a0 = random.uniform(0, 360); a1 = a0 + span
    v = int(120 + 135*random.random())      # 白・明るさ変化のみ
    w = random.choice([2, 2, 3, 3, 4])
    arc(r, a0, a1, max(8, v//5), w+4)        # グロー下地
    arc(r, a0, a1, v, w)                     # 本線
# 中心の極＋極星を描いたあと
img.resize((4096, 4096), Image.LANCZOS).save('public/starfield.png')
```

## 5. ぶつかった問題と解決

- **ぼやける** → 原因は**解像度不足**。本方式は回転で画面全体を覆うため画像を
  「画面の対角線サイズ」まで拡大表示する。最初の画像は 1254×1254 で拡大されて甘くなっていた。
  - 解決: **4096×4096** に。線は **2〜4px相当**で描き拡大に耐えるように。
  - あわせてコード側で 異方性フィルタ・Bloom弱め(0.3)・拡大率を最小化。
  - 目安: 画像は最低でも「画面の対角線px」以上、推奨4096²。PNG推奨（JPGの圧縮ノイズ回避）。
- **静止画を動かしている感**（色が別々だと顕著） → **白一色に統一**して均一感を出し、
  “絵が回ってるだけ感”を軽減。色を散らすと個々の弧が目立ち、回転の作り物っぽさが出やすい。

---

## 7. 学び

- 「回転で全画面を覆う」方式は**解像度が命**。必要画素＝画面対角線以上。
- **同心円＋極中央**にすると回転が数学的に継ぎ目なし（無限ループ可）。
- 色を絞る（白統一）と、ループ／回転の作り物感が下がり**統一感**が出る。
- 動的計算をやめても、**良い素材＋回転＋Bloom**で十分に見栄えがする（軽さと品質の両立）。
