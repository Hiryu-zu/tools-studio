# やりたいことリスト（動く点描 / 光の粒で絵を作る）

最終更新: 2026-06-14
テーマ: ドット（パーティクル）を律動的に動かす「動く点描」。Lv0〜Lv2 をそれぞれ独立ツールとして想定。
土台は既存スタジオ（Three.js + Points + 加算発光 + UnrealBloom）。検討のみ・未実装。

## 前提（重要）
ドットは「描画の手段」であって「絵を考える頭」ではない。
**狙った形を作るにはお手本が要る**（画像・SVG・文字・数式・手描きラフ等）。
画像生成AIなしでも、お手本さえあれば望んだ形を光の粒で再現/構成できる。
（言葉だけで新規の絵をドットが発想するのはAIなしでは不可＝対象外）

---

## Lv0 — 律動する点描（抽象・お手本不要）
**イメージ**: ノイズ場や黄金角配置の点が脈動・流動して有機的に動く、抽象アート。
**実装**: 点を格子/黄金角/乱数で配置 → sin・ノイズ(curl)で揺らす。色も時間で変化。発光＋Bloom。
**難度**: 易。**用途**: 背景演出、アンビエント、ローディング。

## Lv1 — 画像を点描化（静止画 → 動く点描）
**イメージ**: 写真やイラストを読み込み、明暗・エッジに応じて点を配置＝点描画に。
各点を微振動・脈動・流動させ“生きた点描”にする。
**実装**: 画像をオフスクリーンcanvasに描く → 画素をサンプリング（明度/エッジで密度・色を決定）
→ Points に配置 → 各点に小さな揺らぎを加算。Bloomで光の点描。
**難度**: 易〜中。**用途**: ポートレート/風景の幻想化、SNS、ヒーロー背景。

## Lv2 — 点が集まって絵になる（モーフ/集合）
**イメージ**: 散らばった点がお手本の位置へ収束して絵が立ち上がる。崩して別の絵へ移行も可能（画像A→B）。
**実装**: お手本をサンプリングして**目標位置＋目標色**を作る → 点をばね/補間で目標へ移動
→ 目標を差し替えるとモーフ。出現/崩壊（散る）も時間関数で。
**難度**: 中。**用途**: オープニング演出、文字/ロゴの結像、VJ、ストーリーテリング。

---

## 実装メモ（共通）
- Three.js の Points（数千〜十万点）。大量ならGPGPU（テクスチャに位置を持たせて計算）。
- パイプライン: お手本 → 画素サンプリング → 目標位置/色 → 移動(ばね/lerp)＋律動(sin/ノイズ)。
- お手本は画像に限らず SVG・文字・数式/パラメトリック形状でも可（ビットマップ不要の層）。
- 律動・出現・崩壊は「経過時間 t の関数」で設計（この人の作品群で共通の指針）。
- 加算発光は重なると白飛び → Bloom閾値・密度・不透明度で調整。

## 着手するなら（私案）
Lv0（軽くて即映える）→ Lv1（画像点描）→ Lv2（モーフ）の順だと、土台を再利用しながら段階的に作れる。
