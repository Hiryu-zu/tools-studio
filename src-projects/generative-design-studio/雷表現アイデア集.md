# 雷の表現アイデア集（今後やるリスト）

最終更新: 2026-06-14
現在のスタジオには「雷（落雷）」と「高層雷（スプライト／赤い妖精）」を実装済み。
ここに、今後の発展アイデアを「雷のバリエーション」「自然・情景の演出」「ツールとしての需要」の3軸でまとめる。

実装の土台はいずれも共通: **Three.js の加算発光ライン/ポイント ＋ UnrealBloom ＋ ACESトーンマッピング**。
新エフェクトは `src/effects/` に1ファイル足して `registry.ts` に1行追加するだけ。

---

## 0. 実装済み

- ✅ **雷（落雷）** `LightningEffect` … ミッドポイント変位の稲妻＋枝、加算発光＋Bloom、クリック落雷・自動落雷・フラッシュ。
- ✅ **高層雷（スプライト／赤い妖精）** `SpriteLightningEffect` … 上空のクラゲ状放電。赤→紫青のグラデ、淡く幽玄、親雷フラッシュ＋星空。

## 1. 雷そのもののバリエーション（今後）

- [ ] **雲内発光（シート/雲フラッシュ）**: 稲妻が見えず雲全体がボワッと光る。静かで上品。遠雷向き。
  実装: 画面上部に柔らかい発光帯（大きなソフトスプライト）を一瞬光らせるだけ。最軽量。
- [ ] **雲間放電（スパイダー）**: 雲底を横に這う枝分かれ。落雷を横向き＆多分岐にした変種。
- [ ] **リーダー＋リターンストローク**: 暗い先駆放電がツリー状に降り→着地した瞬間に明るい本流が逆走。
  実装: 2段階アニメ（先に全枝を暗く描画→当たった経路だけ強発光を上→下or下→上に走らせる）。リアル度UP。
- [ ] **多重ストローク／余韻**: 同一経路を数回点滅＋落雷後に経路がうっすら残ってフェード。
  実装: strike時に2〜3回の再点滅スケジュール＋残光用の低不透明度パス。
- [ ] **球電（ball lightning）**: 浮遊する発光球。加算スプライト＋ゆらぎ移動。

## 2. 自然・情景として感じさせる演出（今後）

- [ ] **情景に置く（前景シルエット）**: 山/海/草原/地平線のシルエット＋雷。前景があるだけで「自然写真」になる。
- [ ] **遠雷モード**: 遠くで雲がフラッシュ→光がゆっくり広がる。静謐・癒し系。
- [ ] **雷鳴の遅延同期（最有力）**: 光ってから距離に応じて遅れて雷鳴（光速と音速の差＝距離感）。
  実装: Web Audio。閃光時刻＋距離から遅延を計算して再生。「自然を感じる」決定打。
- [ ] **雨・水面との連携**: 雨粒＋既存の水面リップルと合成。閃光で雨が一瞬照らされる／水面に映り込む。
- [ ] **時間帯と色**: 夜の青白い雷、夕暮れのピンク雷、砂嵐のオレンジ。空のグラデと連動。
- [ ] **世界が一瞬見える演出**: 閃光時だけ前景・雲のディテールが浮かび上がる（暗→明のコントラスト）。
- [ ] **カメラの微震／衝撃波**: 落雷時に軽い画面揺れで体感を付与。

## 3. ツールとしての需要・ユースケース（今後）

- [ ] **Web演出素材**: ヒーロー背景・ローディング・404・雰囲気背景。
- [ ] **配信/VJ・オーディオリアクティブ**: 音に反応して落雷。ライブ映像で需要大。
  実装: Web Audio の解析（音量/低音）でstrikeをトリガー。
- [ ] **映像素材の書き出し**: 透過(アルファ)動画として書き出し、編集ソフトで合成。
  実装: MediaRecorder / CCapture。背景透過レンダリング。
- [ ] **教育・可視化**: リーダー/リターンストロークや「光と雷鳴の遅れ＝距離」を体験できる教材。
- [ ] **アンビエント/癒し**: 遠雷＋雨音のループ（ASMR的ライブ壁紙）。
- [ ] **文字・ロゴを雷で描く**: 名前/ロゴ輪郭に沿って放電（リヒテンベルグ的）。SNS・グリーティング向け。

## メモ（実装の勘所）

- 明るい背景や密な加算では **Bloom の threshold を高め**にしないと白飛びする（スプライトで学習済み）。
- 加算発光は重なりで容易に白くなる。**色を残したい時は密度・不透明度を抑える**。
- 透過動画書き出しをするなら、背景を `alpha:true` の透明クリアにしてレンダリング。
