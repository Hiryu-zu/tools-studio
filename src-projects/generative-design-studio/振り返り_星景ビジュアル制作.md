# 振り返り — 星景ビジュアル制作（雛型 → 感動するビジュアルへ）

最終更新: 2026-06-14

水面リップル／星の軌跡の雛型から始まり、最終的に Three.js + ポストプロセスによる
「虹色に光り、奥行きと軌跡のある星景」へ到達するまでの記録。
何をどう変えたか、どう再現したか、抽象的な要望をどんな具体策に落としたかを残す。

---

## 1. 全体の流れ（最初 → 最後）

1. **単体HTMLの雛型**: `index.html` に直書きの Canvas2D / WebGL。水面リップルと星の軌跡。
2. **本格プロジェクト化**: Vite + TypeScript の「Generative Design Studio」へ。共通 Engine＋Effect 差し替え構造に再編。
3. **星の軌跡の改良（2D）**: 点描 → 線分の蓄積＋加算合成＋ビネット。だが「単調・神秘性・遠近感・感動が無い」という壁に当たる。
4. **技術の見直し**: Canvas2D の限界（光の物理表現・奥行きが無い）を確認し、Three.js + postprocessing へ移行を決定。
5. **3D星景の実装**: 加算パーティクル星＋Afterimage（残像＝軌跡）＋UnrealBloom（光の滲み）＋トーンマッピング＋星雲＋距離シェル差速＋虹色。
6. **完成（ひとまず）**: 暗く青い夜空に、近い星は速く長い弧・遠い星はゆっくり短い弧、虹色に発光する星景。

---

## 2. 技術の変遷（最初 vs 最後）

| 項目 | 最初（雛型） | 最後（現在） |
|---|---|---|
| 構成 | 単体HTML直書き | Vite + TypeScript + モジュール構成 |
| 描画 | Canvas2D / 生WebGL1 | Three.js（WebGL）+ EffectComposer |
| 光表現 | なし（平面塗り） | HDR的な加算合成 + UnrealBloom + ACESトーンマッピング |
| 奥行き | なし（2D平面） | 3D空間 + 距離シェル + 透視投影 + 差速 |
| 軌跡 | 2Dの線ストローク蓄積 | AfterimagePass（残像フィードバック） |
| 色 | 白＋わずかな色温度 | 白＋虹色（HSLで色相を全周に散布） |
| UI | 手書きパネル | Tweakpane（パラメータ自動生成・即時調整） |
| 配布 | ファイル共有 | ローカルdev / ビルドして静的ホスティング可 |

**要点**: 「感動するビジュアル」は Canvas2D では構造的に困難だった。
鍵は *光の物理（HDR＋Bloom）* と *奥行き（3D）* で、これらを持つ Three.js + postprocessing に
土台を移したことが最大の転換点。

## 3. 現在の星景ビジュアルの再現方法

エフェクト本体: `src/effects/StarFieldEffect.ts`

### レンダリングの流れ（パイプライン）
1. **シーン**: 暗い青の背景（`clearColor 0x010410`）＋ごく薄いフォグ。
2. **星（加算パーティクル）**: `THREE.Points` を**距離シェル**ごとに生成（近→遠で5層）。
   - マテリアル: `AdditiveBlending` ＋ ソフトな円形スプライト（放射状グラデーションの`CanvasTexture`）＋ `sizeAttenuation`（遠いほど小さく）＋ `depthTest:false`。
   - 近いシェルほど点を大きく、遠いほど小さく。
3. **軌跡**: `AfterimagePass`（前フレームを `damp` で減衰させて重ねる残像）。`damp` が大きいほど弧が長い。
4. **光の滲み**: `UnrealBloomPass`（`threshold` を上げて明るい星だけ発光させ、夜を暗く保つ）。
5. **階調**: `ACESFilmicToneMapping` ＋ `OutputPass`。露出で全体の明るさを調整。
6. **星雲**: 大きな加算スプライト数枚（深い青）でほのかな色と神秘性。
7. **遠近の差速**: 各距離シェルを**別の角速度**で回転（近=速い/遠=遅い）。`spread` で差の強さを調整。
8. **虹色**: 各星の色を一定割合で `HSL(色相=ランダム, 彩度0.85, 明度0.62)` に。`colorful` で割合を調整。

### 主要パラメータ（現在の既定値）
| パラメータ | 値 | 役割 |
|---|---|---|
| 回転速度 speed | 0.06 | 天球の基準角速度 |
| 軌跡(残像) trail(damp) | 0.993 | 大きいほど弧が長い |
| 輝き bloom | 0.70 | Bloomの強さ |
| 露出 exposure | 0.85 | 全体の明るさ |
| 近遠の差速 spread | 0.85 | 0で全シェル同速、1で差最大 |
| 虹色の量 colorful | 0.60 | 0で白のみ、1でほぼ虹色 |
| Bloom threshold | 0.22 | これ以上明るい光だけ滲む（夜を暗く保つ鍵） |
| 距離シェル速度倍率 | 2.4 / 1.7 / 1.2 / 0.85 / 0.55 | 近→遠 |

### 同じ画を作る最短レシピ
> 暗い青背景 → 加算パーティクルの星を距離別レイヤーで配置 → 各レイヤーを別速度で回す
> → AfterimagePassで残像（軌跡）→ UnrealBloomで発光（閾値高め）→ ACESトーンマッピング。
> 色は一部をHSLの虹色に。これで「暗い夜・虹の光・奥行きと軌跡」が同時に出る。

## 4. 抽象的な要望 → 具体的な改善（対応表）

| 要望（抽象） | 何が原因だったか | 具体的にやったこと |
|---|---|---|
| 「神秘性・光の美しさが無い」 | Canvas2Dに発光(HDR/Bloom)が無い | UnrealBloom＋加算合成＋ACESトーンマッピング |
| 「遠近感が無い」 | 2D平面で奥行きが無い | 3D空間化＋距離シェル＋透視投影＋差速 |
| 「単調」 | 色・密度・動きが一様 | 虹色(HSL)＋距離別の密度/速度差＋星雲スプライト |
| 「スタートレイルっぽくない／線が太い」 | 2Dストロークは機械的、残像が短い | 加算パーティクル＋Afterimageの`damp`を上げ、カメラ微動オフで弧を綺麗に |
| 「夜をもっと青く暗く」 | 星雲と閾値0のBloomで白っぽく浮いていた | 背景を `0x010410`、Bloom閾値を上げ、星雲発光と露出を抑制 |
| 「近い光は速く、遠い光はゆっくり」 | 全星を剛体回転（同速）していた | 距離シェルごとに角速度を変える差速回転（`spread`で調整） |
| 「色とりどり・虹を思わせる」 | 白＋わずかな色温度のみ | 一定割合をHSL全色相の高彩度色に（`colorful`で量を調整） |

**学び**: 抽象的な感想（単調・神秘性・遠近感…）は、たいてい特定の技術要素の欠如に1対1で対応する。
「光＝Bloom/HDR」「奥行き＝3D/差速」「彩り＝HSL色相散布」のように言い換えると、具体策に落ちる。

## 5. ハマりどころ・環境メモ（次回の時短のため）

- **Chrome連携は `file://` を開けない**: ローカルHTMLを拡張機能経由で開くと内部ページ扱いで拒否。
  → ローカル確認は **dev サーバ（http://localhost）** 経由にする。
- **サンドボックスは npm レジストリが遮断**（`npm install` が403）: ビルドはここで不可。
  → プロジェクトは生成だけ行い、**ローカルPCで `npm install` / `npm run dev`**。検証は localhost を Chrome連携で見る。
- **フォルダ名に日本語/全角＆があるとViteのファイル監視が効かない**: HMRが反応せず古いモジュールを配信し続けた。
  → `vite.config.ts` で `server.watch.usePolling: true` に。これで毎回の編集が即反映。
- **bashマウントが古いキャッシュを返すことがある**: 整合性確認はファイルツール（Read/Edit/Write）の結果を信頼。
- **ヘッドレスブラウザも入れられない**: 見た目の事前確認は、同じアルゴリズムを Python(PIL) で再現してプレビュー画像化した。
- **AfterimagePassの軌跡長は `damp` で決まる**: e-fold ≒ 1/(1-damp) フレーム。長い弧には 0.99 以上が必要。
- **夜を暗く保ちつつ星を光らせる鍵は Bloomの `threshold`**: 0だと全体が白く浮く。閾値を上げて明るい星だけ滲ませる。

## 6. 現在の構成と起動

```
generative-design-studio/
  src/
    engine/   共通エンジン（ループ・リサイズ・入力・effect切替）・型
    effects/  StarFieldEffect(3D) / StarTrailEffect(2D) / RippleEffect(水面) / registry
    ui/       Controls.ts（Tweakpane）
    main.ts / style.css
  vite.config.ts（usePolling 有効）
```

起動: `npm install` → `npm run dev` → `http://localhost:5173/`
ビルド: `npm run build`（`dist/` を Vercel/Netlify 等に配置可能）
