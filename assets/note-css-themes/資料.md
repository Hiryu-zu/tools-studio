# note用 自作季節テーマCSS 制作依頼まとめ

## 目的

Chrome拡張機能 **Stylus** などで利用するための、`note.com` 向け自作CSSテーマを作成したい。

目的は、noteの見た目を自分専用の作業環境として擬似的に変更すること。
公開されるnote記事の見た目を変えるのではなく、**自分のブラウザ上だけで、noteを創作机・読書室・思考ノートのように演出する**。

---

# 前提

## 対象

* 対象サイト：`note.com`
* 主な利用環境：PC版Chrome
* 想定拡張機能：Stylus
* スマホアプリ版noteは対象外
* スマホ版Chromeも基本的に対象外

## 重要な制約

* noteアプリ自体の見た目は変更できない前提
* 読者側のnote表示は変わらない
* CSSは自分のブラウザ内だけに適用される
* 画像やサムネイルの色はなるべく変えない
* 記事本文や投稿画面が読みづらくならないことを優先する
* note側の仕様変更で崩れる可能性があるため、CSSはなるべく汎用的にする

まず以下の3種類を作りたい。

1. 薄紅色の桜柄テーマ

3. 水色テーマ

1. 春：桜文机
2. 夏：水面ノート
3. 秋：紅葉読書室
4. 冬：雪灯り

CSSだけで難しい場合は、CSS内SVG、または画像背景の利用も検討する。

---

# 実装方針

## 優先順位

```text
第1段階：CSSだけで色と淡い模様を作る
第2段階：CSS内SVGで桜・紅葉・雪輪などを表現する
第3段階：画像背景を使って、より自由な季節テーマにする
```

最初はCSSのみで実装し、背景パターンは `radial-gradient` や `linear-gradient` を使って軽量に作る。
模様が抽象的すぎる場合は、CSS内にSVGを埋め込む形を検討する。

---

# 共通CSS設計

## CSS変数を使う

各テーマでは、以下のようなCSS変数を使いたい。

```css
:root {
  --bg-main: #ffffff;
  --bg-paper: #ffffff;
  --bg-soft: #f5f5f5;
  --text-main: #222222;
  --text-sub: #666666;
  --accent: #888888;
  --accent-soft: #dddddd;
  --border-soft: #eeeeee;
}
```

## 主な変更対象

* `html`
* `body`
* `main`
* `article`
* `section`
* `div[role="main"]`
* `h1, h2, h3, h4`
* `p, span, li`
* `a`
* `button`
* `textarea`
* `input`
* `[contenteditable="true"]`
* `hr`

## 基本方針

* 背景は淡くする
* 文字色は十分なコントラストを保つ
* 本文エリアは白系・紙色系で読みやすくする
* リンク色はテーマカラーに合わせる
* 入力欄や投稿画面も違和感なく調整する
* 画像・動画・サムネイルには基本的にフィルターをかけない

# 画像背景を使う場合

CSSだけで模様表現が難しい場合は、画像背景を使う。

## 例

```css
body {
  background-color: #fff7f8 !important;
  background-image: url("画像URL");
  background-repeat: repeat;
  background-size: 240px 240px;
  background-attachment: fixed;
}
```

## 注意点

* ローカル画像を直接指定すると、ChromeやStylusでうまく読めないことがある
* Web上に画像を置いてURL指定する方が扱いやすい
* ただし公開URLにすると画像が外部から見える可能性がある
* 個人用なら、CSS内SVGの方が管理しやすく安全
* 画像背景は重くなりすぎないようにする
* 背景模様は淡く、本文の読みやすさを邪魔しないこと

---

# CSS内SVGを使う場合

## 方針

外部画像ファイルを使わず、CSS内にSVGを埋め込む。

メリット：

* 画像ファイルを別管理しなくてよい
* Stylus内だけで完結する
* 外部URLを使わなくてよい
* パターン背景として使いやすい

デメリット：

* SVGコードが長くなる
* 複雑な柄は作りにくい
* 編集がやや面倒

## 向いている模様

* 桜の花びら
* 紅葉の葉
* 雪輪
* 青海波
* 波紋
* 小さな花
* 和紙風の粒

## 依頼1：note.com用CSSを作成する

---

## 依頼2：CSSだけ版を優先する

まずは外部画像なしで、CSSだけで動く版を作成する。

使ってよいもの：

* CSS変数
* `radial-gradient`
* `linear-gradient`
* `repeating-linear-gradient`
* `background-size`
* `background-position`
* `body::before`
* `opacity`
* `mix-blend-mode` は必要な場合のみ
* CSS内SVG

---

## 依頼3：本文の読みやすさを最優先する

装飾よりも可読性を優先する。

必須条件：

* 本文が読みやすい
* 背景模様が主張しすぎない
* 文字色が薄すぎない
* 入力欄が読める
* ボタンやリンクが見える
* noteの画像・サムネイルの色は変えない
* 全体が重くなりすぎない

---

## 依頼4：コメントを多めに入れる

あとで自分で編集できるように、CSS内にコメントを入れてほしい。

例：

```css
/* 背景色 */
--bg-main: #fff7f8;

/* 本文カード色 */
--bg-paper: #fffdfd;

/* リンク色 */
--accent: #d9899b;

/* 模様の濃さ */
opacity: 0.45;
```

---

## 依頼5：調整しやすい構造にする

色や模様の濃さを後から変更しやすくしたい。

特に変更しやすくしたい値：

* 背景色
* 本文エリア色
* 文字色
* リンク色
* 枠線色
* 模様の濃さ
* 模様のサイズ
* 模様の位置

# まとめ

このプロジェクトの目的は、noteを自分専用の創作環境に変えること。

単なる色変更ではなく、以下のような感覚を目指す。

* 春は桜の文机
* 夏は水面のノート
* 秋は紅葉の読書室
* 冬は雪灯りの書斎

CSSだけで可能な範囲から始め、必要に応じてCSS内SVGや画像背景を使って、季節ごとの作業空間を作っていきたい。
