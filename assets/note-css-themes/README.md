# note.com 自作季節テーマCSS

Chrome拡張 **Stylus** で使う、`note.com` 専用の自作テーマです。
自分のブラウザ上だけでnoteの見た目を変え、季節ごとの「創作机・読書室」のように演出します。
公開記事の見た目や、他の読者の表示は一切変わりません。

## テーマ一覧

| ファイル | テーマ | 模様（初期表示） | 向いている用途 |
|---|---|---|---|
| `note-theme-pink.css` | 薄紅色の桜柄（基本） | 桜の花（5弁） | 創作・詩・文学系・やわらかい執筆 |
| `note-theme-lightgreen.css` | 薄緑色（基本） | 若葉（葉脈つき） | 読書メモ・学習・思考整理・長文 |
| `note-theme-blue.css` | 水色（基本） | 波（水面のうねり） | 技術記事・AIメモ・設計書・コード学習 |
| `note-theme-autumn.css` | 秋：紅葉読書室 | 紅葉（もみじ・赤強め） | 読書メモ・文学的問い・レビュー・内省 |
| `note-theme-winter.css` | 冬：雪灯り | 六花（雪の結晶）と白椿 | 静かな創作・神秘系メモ・内省 |
| `note-focus-mode.css` | 執筆集中モード（共通） | 模様を消して無地に | どのテーマにも重ねてON/OFF |

`preview.html` をブラウザで開くと、全テーマの見た目をモック画面で切り替えられ、
右上の「✎ 執筆集中モード」ボタンで集中モードのON/OFFも試せます。

## このバージョンの変更点

- **色をより鮮やか**にしました（リンク・見出し・アクセントの彩度を上げ、背景もやや色味を強めに）。本文の文字色は濃いまま、可読性は維持しています。
- **模様を具体的な絵柄（CSS内SVG）に**しました。桜の花・若葉・波（水面）・紅葉（もみじ）・六花と白椿を、各テーマで初期表示にしています。水色は波らしいうねり、秋は楓の葉だけで赤みを強め、冬は六花と白椿のみに整理しました。
- **執筆集中モード（`note-focus-mode.css`）を別ファイル**として追加。季節テーマと重ねて、ワンタッチでON/OFFできます。

## 導入手順（Stylus）

1. ChromeにStylusをインストールする。
2. Stylusのアイコン →「新しいスタイルを書く」。
3. 使いたいテーマの `.css` を全文コピーして貼り付ける。
4. 「適用先」で「次で始まるURL」→ `https://note.com` を指定する。
5. 名前を付けて保存。
6. 複数テーマを登録しておき、その日の気分で有効なものを1つだけONにする。

## 執筆集中モードの使い方

1. `note-focus-mode.css` を **季節テーマとは別のスタイル**としてStylusに新規登録する（適用先は同じ `note.com`）。
2. 季節テーマは有効のままで構いません。集中モードを **ON にすると模様が消えて無地の紙になり、本文が広く・行間ゆったり**になります。**OFF にすると通常の季節表示**に戻ります。
3. 行間や紙幅はファイル冒頭の `--focus-line` や「本文カードの横幅」ブロックで調整できます。

集中モードは後から読み込ませて上書きする設計（`!important`多め）なので、どの季節テーマと組み合わせても効きます。

## 模様の「具体的SVG版」と「抽象CSS版」の切り替え

各テーマの `body::before` には **【A】具体的SVG版**（初期有効）と
**【B】抽象CSS版**（軽量・コメントアウト状態）の2つが入っています。
軽くしたい・絵柄を抑えたいときは【A】を丸ごとコメントアウトし、【B】のコメントを外してください。
有効な `body::before` は必ずどちらか一方だけにします。

## 調整のしかた

各ファイル冒頭の `:root` 変数を書き換えるだけで変わります。

```css
--bg-main:    背景色（ページ全体）
--bg-paper:   本文カード色（記事エリア）
--text-main:  本文の文字色（薄くしすぎ注意）
--accent:     リンク・アクセント色
--border-soft:枠線色
--pattern-opacity: 模様の濃さ（0で完全に消える）
--pattern-size:    模様タイルの大きさ
```

- **模様を薄くしたい** → `--pattern-opacity` を `0.2` などに下げる
- **模様を完全に消したい** → `--pattern-opacity: 0;`

## 読みやすさについて

全テーマとも本文カード（`article` / `main` / `section`）は紙色でほぼ不透明にしてあり、
模様は背景にだけ薄く出ます。文字色はコントラストを確保した濃さです。
画像・動画・サムネイルにはフィルターをかけていないので、写真の色は変わりません。

## note側の仕様変更で崩れたときの調整ポイント

- 背景模様が本文の前面に出る → `body::before` の `z-index: -1;` と `inset: 0;` を確認。
- 本文が透けて読みにくい → `article, main, section` の `background-color` を不透明な色に。
- 一部の文字色が変わらない → その要素のセレクタを `p, span, li` の並びに追加。
- 入力欄が暗い／読めない → `textarea, input, [contenteditable="true"]` のブロックを確認。
- 集中モードで紙幅がおかしい → `note-focus-mode.css` の「本文カードの横幅」ブロックを削除。

## 注意

- PC版Chrome + Stylus を想定（スマホアプリ版note・スマホ版Chromeは対象外）。
- noteアプリ自体の見た目は変更できません。自分のブラウザ内だけの演出です。
- 外部画像は使っていません（SVGはCSS内に埋め込み）。
